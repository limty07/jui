Log

JUI 라이브러리는 화면에 생성된 컴포넌트 객체에 대한 기본적인 정보를 볼 수 있는 도구를 제공하며, 컴포넌트 객체의 메소드가 호출되는 시점을 실시간으로 체크하여 메소드 실행 시간과 같은 부가적인 정보도 알 수 있습니다.

Summary

현재 페이지에서 생성된 컴포넌트들에 대한 기본적인 정보를 보여주는 탭입니다. UI Caller는 컴포넌트 생성 메소드를 호출한 횟수이며, 실제로 생성된 객체 수는 UI Object를 통해 알 수 있습니다. 좀 더 설명을 하자면 tooltip_1 아이디를 가진 마크업이 하나이기 때문에 UI Caller와 UI Object 개수가 동일하게 나옵니다. 

<span id="tooltip" title="Tooltip Message">Run</span>
tooltip = ui.tooltip("#tooltip");
tooltip.show();

하지만 아래 코드와 같은 경우에는 tooltip 클래스를 가진 마크업이 하나가 아닌 두 개가 존재하기 때문에 UICaller는 1개가 되며, UI Object는 2개가 됩니다. 

<span class="tooltip" title="Tooltip Message 1">Run</span>
<span class="tooltip" title="Tooltip Message 2">Run</span>
tooltip = ui.tooltip(".tooltip");
tooltip[0].show();
tooltip[1].show();

그 밖에 DOM 이벤트와 JUI에서 제공하는 커스텀 이벤트가 얼마나 설정되어 있는지를 알 수 있고, 각각의 컴포넌트 별로 생성된 객체 수를 알 수 있습니다.

Information

컴포넌트 생성 타입과 옵션 등의 자세한 정보를 볼 수 있습니다. 그리고 Obj Cnt는 앞서 설명한 UI Object의 개수입니다.

Log

UI Object의 공개된 메소드를 호출할 경우에 실시간으로 보여지는 로그입니다. 주로 어떤 메소드가 호출되었으며 얼마만큼의 실행 시간이 걸렸는지 알 수 있습니다.
