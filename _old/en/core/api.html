API

The JUI library provides a global object jui. Each component is created using this object.

Before the ready point

JUI provides the following methods and properties for use prior to the ready point.
All of the components in the JUI can be created once the callback function of ready has been called.

Name Type Description
ready method Method provided for use of a component module.
defineUI method Method that defines a component module.
define method Method that defines a class or object which is heritable
defineOptions method Gets options defined in a module in a heritable structure and checks option validity.
include method Gets a class or object identified as 'define' or 'defineUI'
includeAll method Gets all classes and objects that are identified as 'define' or 'defineUI'
log method Method that displays a pop-up on which the user can view the log.
setup method Configures the basic settings of the JUI library, for example the log URL and template grammar settings.

After ready

Name Parameter Return Description
add uiObj none Adds a component object created.
emit uiType | uiSelector, eventType, eventArgs none Generates a custom event to an applicable component
get index | uiType | uiSelector object | array Brings a component currently created
getAll none array Brings all components currently created
remove index uiObj Removes a component object in an applicable index from the list
shift none uiObj Removes the last component object from the list.
pop none uiObj Removes the first component object from the list.
size none int Gets the number of objects currently created
addClass uiCls none Adds a component class
getClass index or uiType object or array Gets a component class
getClassAll none array Gets all component classes
create uiType, selector, options object It is possible to create a component dynamically after the ready point.
debugAll callback none debugs all component objects currently existing
