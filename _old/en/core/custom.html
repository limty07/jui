Custom

JUI library provides a send box method that facilitates the development of UI components. There are two types of UIs that can be added. A file is created in the js/ui or js/uix folder depending on the characteristics of the new component. That is to say, a ui component is not dependent on other components, while a uix component is created using combinations of other components.

Define

Assuming a component file exists in the js/ui or js/ui folder, use the method as follows. 

jui.defineUI("ui.test", [ "util.base" ], function(_) {
	var UI = function() {
		this.init = function() {
			// initially implemented part
		}
		
		this.func1 = function(val) {
			// open method 1
		}
	}

    UI.setup = function() {
        return { // default UI option settings
			option1: 1000,
			option2: true
        }
    }
	
	return UI;
}, "core");

The setting imethod defines the option(s) to be used in connection with a component, and init is a method that is initially run when a component object is created. These two methods must be defined when adding a new UI component.

If a user wants to add a new uix component that uses the dropdown component, configure the component as follows; 

jui.defineUI("uix.test", [ "ui.dropdown", "util.base" ], function(dropdown, _) {
	var UI = function() {
		// same as above
	}
	
	return UI;
}, "core");

core

In the class returned by the define method, the basic methods and properties required by a UI component can be called using this. For example, a custom method can be created with the edit method. If you are using the applicable component you can listen for a custom event using the on method or the component event option.
